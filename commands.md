## Commands

| Command | Description | Typical Use |
| --- | --- | --- |
| `/moderation warn` | Issues warning actions where configured. | Behavior management |
| `/moderation mute` | Applies mute/moderation restrictions. | Incident response |
| `/moderation unmute` | Removes active mute/restriction. | Resolution |
| `/moderation ban` | Executes ban action with logging. | High-severity moderation |
| `/moderation unban` | Reverses ban when needed. | Appeals/reinstatement |
| `/automation rules` | Shows active automation rules. | Rule audit |
| `/automation enable` | Enables a configured automation module. | Feature rollout |
| `/automation disable` | Disables automation module. | Safe rollback |
| `/logs channel` | Sets moderation/action log destination. | Visibility setup |
| `/settings roles` | Reviews role-based access configuration. | Permission model |
| `/help` | Shows command categories and quick navigation. | Discovery |
| `/support` | Returns support resources. | Get help |

> Live command names can vary by deployment; use `/help` in-server as source of truth.
